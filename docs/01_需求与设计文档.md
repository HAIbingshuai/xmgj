# 项目管家 - 需求与设计文档

> 最后更新：2026-05-29 | 当前版本：v8

---

## 一、项目概述

**项目名称：** 项目管家
**技术栈：** Django 4.2.30 + Bootstrap 5 + SQLite + Bootstrap Icons
**核心定位：** 个人事项常态/长时间跟进追踪系统
**用户模式：** 多用户（需要登录），数据隔离
**设计原则：** 由内到外（先做核心详情页，再向外扩展）
**视觉稿版本记录：** `design_mockup/detail_page_v1.html` ~ `v8.html`（最新为 `detail_page.html` = v8）

## 二、数据模型设计

### 2.1 Item（事项）

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | CharField(200) | 标题 |
| `description` | TextField | 描述/概述 |
| `status` | CharField(20) | 状态：未开始/进行中/已完成/暂停/已归档 |
| `priority` | CharField(10) | 优先级：低/中/高/紧急 |
| `creator` | ForeignKey(User) | 创建人（数据隔离用） |
| `created_at` | DateTimeField | 创建时间 |
| `updated_at` | DateTimeField | 更新时间 |

**排序：** 按更新时间倒序

### 2.2 TimelineNode（时间轴节点）

| 字段 | 类型 | 说明 |
|------|------|------|
| `item` | ForeignKey(Item) | 所属事项 |
| `sequence` | PositiveIntegerField | 序号（同一事项内唯一） |
| `title` | CharField(200) | 节点标题 |
| `summary` | TextField | 节点概要 |
| `node_date` | DateField | 节点日期 |
| `status` | CharField(20) | 状态：待处理/进行中/已完成/已跳过 |

**排序：** 按日期+序号升序
**唯一约束：** (item, sequence)

### 2.3 NodeStep（节点步骤）

| 字段 | 类型 | 说明 |
|------|------|------|
| `node` | ForeignKey(TimelineNode) | 所属节点 |
| `sequence` | PositiveIntegerField | 步骤序号 |
| `content` | TextField | 步骤内容（描述） |
| `status` | CharField(20) | 状态：未开始/进行中/已完成/受阻 |
| `note` | TextField | 备注/结果记录 |

**唯一约束：** (node, sequence)

### 2.4 NodeStepFile（步骤文件）

| 字段 | 类型 | 说明 |
|------|------|------|
| `step` | ForeignKey(NodeStep) | 所属步骤 |
| `file_type` | CharField(10) | 类型：attachment(附件) / image(图片) |
| `file` | FileField | 文件 |
| `original_name` | CharField(255) | 原始文件名 |
| `file_size` | IntegerField | 文件大小(bytes) |
| `created_at` | DateTimeField | 上传时间 |

### 2.5 NodeAction（操作记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| `node` | ForeignKey(TimelineNode) | 所属节点 |
| `content` | TextField | 操作内容 |
| `action_type` | CharField(20) | 操作类型（见下方） |
| `operator` | ForeignKey(User, SET_NULL) | 操作人 |
| `created_at` | DateTimeField | 操作时间 |

**操作类型（4色分类）：**
| 类型 | 含义 | 圆点颜色 |
|------|------|---------|
| `create` | 创建节点/添加步骤 | 蓝色 `#4A6FA5` |
| `update` | 更新内容/添加备注 | 琥珀色 `#F59E0B` |
| `complete` | 完成步骤/状态完成 | 绿色 `#28A745` |
| `problem` | 状态变更为受阻/问题 | 红色 `#DC3545` |

**排序：** 按时间倒序（最新的在前面）

### 3.3 垂直时间轴（左栏）

**视觉规范：**
- 垂直线：2px `#E9ECEF`，圆点左侧 14px
- 圆点：直径 18px，带 3px 白色外圈（box-shadow），hover scale(1.25)

**状态对应颜色：**
| 状态 | 圆点填充 | 边框 |
|------|---------|------|
| 待处理(pending) | 白色 | `#ADB5BD` |
| 进行中(in_progress) | `#FFC107` | `#FFC107` |
| 已完成(completed) | `#28A745` | `#28A745` |
| 已跳过(skipped) | `#6C757D` | `#6C757D` |

**选中态增强：**
- 圆点放大至 24px，边框加粗至 4px
- 对应状态的发光 shadow（如进行中：`0 0 12px rgba(255,193,7,0.4)`）
- 卡片左侧弹出蓝色三角指示器
- 卡片蓝色边框 + 外发光 + 背景变白

**节点卡片内容：** 日期 → 标题 → 概要(1行) → 步骤进度 3/3 → 迷你进度条(3px)

### 3.4 节点详情面板（右栏）

**构成顺序：**
1. 节点头部：标题 + 状态徽章 + 日期 + 操作按钮群(🕐操作记录/＋添加步骤/✏️编辑/⋮更多)
2. 概要描述文字
3. 步骤进度条
4. 步骤清单（标题栏含「+ 添加」快捷按钮）
5. 每个步骤 = 折叠头 + 展开体

### 3.5 步骤卡片

**折叠态（默认）：**
```
☑ 确定设计规范与风格定调          [已完成]
   📎2  🖼️3                       ← 有附件/图片时显示
   详细描述预览文字...              ← 内容预览
```

**展开态：**
```
☑ 确定设计规范与风格定调          [已完成]  [▼]
   ──────────────────────────────────────
   详细描述文字...

   ✓ 结果: 设计规范已确认，全部定稿。   ← 结果区（上移）

   [编辑] [附件(2)] [图片(3)]          ← 按钮栏（固定底部）
```

**结果区样式：**
| 状态 | 样式 |
|------|------|
| 正常完成 | 绿色背景 `#F0FDF4` + `#28A745` 图标 |
| 进行中 | 琥珀色背景 `#FFF9ED` + `#F59E0B` 图标 |
| 未开始 | 灰色背景 + 斜体 "尚未开始" |
| 受阻 | 红色背景 `#FEF2F2` + `#DC3545` 图标 |

**按钮条件显示规则：**
| 情况 | 按钮 |
|------|------|
| 有附件 + 有图片 | [编辑] [附件(n)] [图片(n)] |
| 有图片无附件 | [编辑] ~~附件 —~~ [图片(n)] |
| 无附件无图片 | [编辑] ~~附件 —~~ ~~图片 —~~ |

灰色不可点按钮 `—` 占位保持布局稳定。

## 四、路由设计

```
/                           → 仪表盘
/items/                     → 事项列表
/items/create/              → 新建事项
/items/<id>/                → ★ 事项详情（核心页面）
/items/<id>/edit/           → 编辑事项
/items/<id>/delete/         → 删除事项
/items/<id>/nodes/create/   → 添加时间轴节点
/items/<id>/nodes/<nid>/    → 节点详情（AJAX面板/全页）
/items/<id>/nodes/<nid>/edit/   → 编辑节点
/items/<id>/nodes/<nid>/delete/ → 删除节点
/items/<id>/nodes/<nid>/steps/create/         → 添加步骤
/items/<id>/nodes/<nid>/steps/<sid>/update-status/ → AJAX更新步骤状态
/auth/login/                → 登录
/auth/register/             → 注册
/auth/logout/               → 退出
```

## 七、v8 审查问题清单（待开发前确认）

### 问题 1：点击 [编辑] 按钮后的状态变化（已确认）

**方案：行内编辑**
- 描述(text) → 原地变为 textarea
- 结果(text) → 原地变为 textarea
- 按钮变化：[编辑] → [保存] + [取消]
- 附件(n)/图片(n) 按钮在编辑模式下保留

### 问题 2：右上角「⋮ 更多」菜单内容（已确认）

**菜单项：**
- 编辑节点（跳转编辑页，同 ✏️ 功能）
- 调整排序（弹窗拖动排序）
- 删除节点（红色，弹窗确认）

### 问题 3：删除操作的确认流程（已确认）

**方案：全部弹窗确认**
- 删除节点 → 弹窗确认
- 删除步骤 → 弹窗确认
- 删除附件 → 弹窗确认
- 删除图片 → 弹窗确认

### 问题 4：时间轴日期跨年显示

**现状：** 左栏日期显示 "05-20"，跨年场景（2025-12 → 2026-01）可能混淆。
**方案：** 默认显示 "05-20"，hover tooltip 显示完整 "2026-05-20"；或同一事项内跨年时自动补全年份。

### 问题 5：折叠态 meta 信息为空时

**现状：** 步骤既无附件也无图片时，折叠态 meta 行空白，看起来信息缺失。
**方案：** 无附件/图片时不显示 meta 行，或只显示步骤号。

### 问题 6：加载状态 / 错误状态

**现状：** 上传无进度、保存无反馈、失败无提示。
**方案：** 开发时预留 loading spinner、上传进度条、错误 toast 位置。

### 问题 7：中等屏幕（992-1200px）适配

**现状：** 30% 左栏在 <1200px 时过窄。
**方案：** 992-1200px 范围内适当缩小内边距，或调整左右比例为 35/65。

### 问题 8：可访问性

**现状：** 缺少 focus 样式，键盘 Tab 无焦点指示。
**方案：** 开发时加入 `:focus-visible` 样式。
