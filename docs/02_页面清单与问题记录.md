## 5. ★ 事项详情页 `/items/<id>/`（核心）

**功能：** 垂直时间轴 + 节点详情 + 步骤管理 + 附件/图片/操作记录

**布局：**
```
┌─ Navbar ──────────────────────────────────────┐
├─ [←返回] │ 事项标题 [进行中] [高] 描述...     ┤
├──────────────────────┬────────────────────────┤
│  时间轴 [✏️][＋]     │  节点标题 [进行中]      │
│                      │  2026-05-25 [🕐][＋][✏️][⋮]│
│  ● 05-20 节点A [完成]│                        │
│  │ 步骤 3/3 ████    │  步骤进度 3/5 ██████    │
│  │                   │                        │
│  ● 05-25 节点B [进行]│  ─── 步骤清单 ── [+添加]│
│  │ 步骤 3/5 ████    │                        │
│  │                   │  ☑ 步骤内容 [已完成]    │
│  ● 06-05 节点C [待]  │  │ 步骤内容文字...      │
│  │ 步骤 0/4          │  │ ✓ 结果文字           │
│  │                   │  │ [编辑][附件(2)][图片] │
│  ● 06-15 节点D [待]  │                        │
│  │ 步骤 0/2          │  ☑ 步骤内容 [进行中]... │
│                      │                        │
│  (左栏 30%)          │  + 添加步骤             │
│                      │  (右栏 70%)             │
└──────────────────────┴────────────────────────┘
```

**抽屉：**
| 抽屉 | 方向 | 内容 |
|------|------|------|
| 附件管理 | ← 左侧滑出 | 上传区 + 文件列表(下载/删除) |
| 图片管理 | ← 左侧滑出 | 上传区 + 缩略图网格(删除) |
| 操作记录 | → 右侧滑出 | 时间线日志(4色圆点) |

**当前问题：**
- [ ] **首次进入详情页时，默认选中第一个节点，但左侧圆点和卡片没有正确显示 `.selected` 类的高亮效果** —— 检查 HTML 确认 `node == selected_node` 比较逻辑
- [ ] **`node_card.html` 中的选中判断**：`{% if node == selected_node %}` 在 Django 模板中比较 QuerySet 对象可能因为数据库重新查询而失效，应使用 `pk` 比较：`{% if node.pk == selected_node.pk %}`
- [ ] **右侧详情面板的步骤卡片**在 AJAX 加载后，`step-header` 的点击展开事件需要重新绑定，`main.js` 里的 `item_detail.html` 中 `extra_scripts` 已经做了处理，但需要验证是否覆盖所有情况
- [ ] **步骤卡片展开默认逻辑**：`{% if step.status == 'in_progress' %}expanded{% endif %}` — 确认这是否合理（用户可能希望所有步骤都折叠，或仅展开第一个）
- [ ] **附件抽屉的 upload-zone**：点击上传后整个表单提交（刷新页面），用户体验不够好。但暂时可用
- [ ] **图片抽屉**：点击上传图片后刷新页面，但新上传的图片不会立即出现在抽屉里（需要关掉重开）
- [ ] **结果区的图标 `style` 属性**在 step_card.html 中有嵌套引号问题（jinja2 的 `{% %}` 标签内 `style="..."` 中的双引号冲突），可能导致图标不渲染
- [ ] **⋮ 更多菜单**中"调整排序"尚未实现功能
- [ ] **没有步骤时**：右侧面板显示"暂无步骤"和"添加第一个步骤"按钮，但左栏仍显示 0/0 的步骤进度，看起来不协调
